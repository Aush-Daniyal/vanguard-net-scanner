Асинхронная система сетевого аудита и сканирования портов с графическим интерфейсом на PyQt6.


Ядро построено на `asyncio` и низкоуровневых сокетах (`asyncio.open_connection`), что позволяет параллельно проверять тысячи портов в секунду. После обнаружения открытого порта система пытается захватить баннер сервиса (HTTP, SSH, FTP, SMTP и др.) и сравнивает его с встроенной базой сигнатур известных уязвимостей, выставляя оценку критичности: **Low / Medium / High**.

Интерфейс выполнен в минималистичном тёмном стиле, вдохновлённом веб-интерфейсом Gemini: глубокий угольный фон, скруглённые карточки, акцентный синий цвет и плавная подача результатов в реальном времени без зависания окна (сканирование выполняется в отдельном `QThread`, не блокируя GUI).

## ⚠️ Важно перед использованием

Сканирование портов чужих хостов без явного разрешения владельца — нарушение закона в большинстве стран (в РФ — статьи 272/273 УК). Используйте Vanguard-Net только для:

- аудита собственной инфраструктуры (домашняя сеть, личные серверы, VPS);
- тестирования систем, на которые у вас есть письменное разрешение (договор pentest, bug bounty с явным scope).

Встроенная база сигнатур в `analyzer.py` — демонстрационный набор паттернов по баннерам и номерам портов, а не полноценная синхронизированная база CVE/NVD. Для продуктивного аудита её нужно дополнять актуальными данными.

Требуется Python 3.10+.

```bash

```

## Использование

1. **Target IP / Hostname** — адрес цели (например, `127.0.0.1`, `192.168.1.1`, `myserver.local`).
2. **Port Range** — диапазон портов в формате `1-1024`, либо список через запятую `22,80,443`, либо смешанный `22,80,8080-8090`. Пустое поле = диапазон `1-1024`.
3. **Таймаут соединения** — слайдер от 0.2 до 5.0 секунд на одну попытку подключения. Увеличивайте при сканировании удалённых/медленных хостов, уменьшайте для localhost.
4. **Параллельные потоки** — слайдер от 50 до 2000, количество одновременных TCP-подключений (через `asyncio.Semaphore`).
5. Нажмите **«Начать сканирование»**. Результаты появляются карточками в реальном времени по мере нахождения открытых портов:
   - 🟥 пастельно-красная подсветка — найдена сигнатура высокой критичности (High);
   - 🟨 жёлтая — средняя критичность (Medium);
   - 🟩 пастельно-зелёная — порт открыт, явных признаков уязвимости не найдено (Info/Low).
6. Кнопка **«Остановить»** прерывает сканирование, не дожидаясь завершения всех портов.

## Как проверить, что сканер находит порты достоверно

Самый надёжный способ — сравнить с независимым эталонным инструментом и с собственным контролируемым тестовым сервером.

### Сравнение с nmap

```bash
nmap -p 1-1024 127.0.0.1
```

Просканируйте тот же диапазон в Vanguard-Net на той же машине. Список открытых портов должен совпасть. Расхождения обычно означают, что нужно увеличить таймаут (порт не успевает ответить) — это не повод считать сканер неисправным сам по себе, но повод перепроверить настройки.

### Проверка на собственном тестовом сервере

```bash
python -m http.server 8765
```

Оставьте терминал с сервером открытым — `http.server` логирует каждый входящий запрос. Просканируйте диапазон, включающий `8765` (например, `8760-8770`). Вы должны увидеть:

- в Vanguard-Net: порт 8765 отмечен как открытый, баннер содержит `HTTP/1.0 200 OK` и строку `Server: SimpleHTTP/...`;
- в терминале с `http.server`: новую строку лога вида `"HEAD / HTTP/1.1" 200 -`, появляющуюся в момент скана — это прямое подтверждение, что запрос реально дошёл до сервера.

Для сверки баннера вручную:

```bash
curl -I http://127.0.0.1:8765
```

Ответ должен совпадать по смыслу с тем, что показала карточка в интерфейсе.

### Проверка ложноположительных срабатываний

Просканируйте диапазон портов, в котором вы точно уверены, что ничего не запущено (например, `54000-54010` на чистой машине). Результат должен быть «открытых портов не найдено». Если сканер показывает там что-то открытым — это баг, а не особенность сети.

### Проверка сигнатурного анализатора

Поднимите сокет, отправляющий заведомо уязвимый баннер:

```python
import socket
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
s.bind(('127.0.0.1', 9002))
s.listen(1)
print("Test server running on 9002")
while True:
    conn, _ = s.accept()
    conn.send(b"220 (vsFTPd 2.3.4)\r\n")
    conn.close()
```

Просканируйте порт 9002 — карточка должна выйти **красной (High)** с указанием на известный backdoor vsFTPd 2.3.4 (CVE-2011-2523). Это подтверждает, что regex-сигнатуры в `analyzer.py` реально применяются к полученным баннерам, а не являются мёртвым кодом.

## Технические детали реализации

- **Конкурентность**: `asyncio.Semaphore` ограничивает число одновременных TCP-подключений значением со слайдера, чтобы не упереться в лимиты файловых дескрипторов ОС при больших диапазонах портов.
- **Banner grabbing**: сначала пассивная попытка чтения (сервис сам присылает баннер — характерно для SSH, FTP, SMTP), затем активные пробы для известных протоколов (HTTP HEAD-запрос, SMTP EHLO и т.д.), и generic HTTP-fallback для портов, не входящих ни в один известный список — это покрывает нестандартные веб-порты вроде 8765.
- **Поток выполнения**: GUI-поток (Qt event loop) никогда не блокируется — всё сканирование выполняется в `QThread`, который внутри себя поднимает собственный `asyncio` event loop через `asyncio.run()`. Связь между потоками идёт через `pyqtSignal`, что безопасно с точки зрения Qt-потоковой модели.
- **Сигнатурный анализ**: комбинация двух механизмов — regex-паттерны по содержимому баннера и статическая таблица «высокорисковых» номеров портов (срабатывает даже без специфичного баннера, например для Telnet, RDP, открытого Docker API).

## Известные ограничения

- База уязвимостей — демонстрационный, не исчерпывающий набор сигнатур.
- Banner grabbing для TLS-портов (443, 8443) не выполняет полноценный TLS handshake — отображается информационная заглушка.
- Инструмент не выполняет UDP-сканирование, только TCP.
